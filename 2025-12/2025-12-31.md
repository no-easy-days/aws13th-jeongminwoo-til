1. 리스트(List): 동적 배열 방식
리스트는 메모리의 연속된 공간을 확보해서 데이터를 순서대로 채운다. 
- 저장 방식: 인덱스 0번, 1번, 2번... 순서대로 빈틈없이 데이터를 배치한다.


2. 셋(Set): 해시 테이블 방식
셋은 연속된 순서가 아니라, 해시 함수가 지정해준 무작위 위치에 데이터를 저장한다.

- 저장 방식:  
  1. 데이터를 넣기 전 해시 함수에 데이터를 먼저 통과시킨다.  
  2. 나온 결과값을 인덱스 번호로 삼아 그 위치에 데이터를 던져 넣는다.

## 해시 테이블(Hash Table)
- 해시 테이블은 (Key, Value)로 데이터를 저장하는 자료구조 중 하나이다.

### 해시 함수(Hash Function)
- 해시 함수는 입력받은 값(문자열, 숫자 등)을 고정된 길이의 숫자로 변환해주는 함수이다.
 
## 시간 복잡도
- 시간 복잡도는 데이터가 많아질 때 알고리즘이 실행되는 시간이 얼마나 늘어나는가를 나타내는 척도이다.  
Big-O 표기법 : O(1), O(n) - 아무리 최악의 상황이라도 이 정도 시간 안에는 끝난다라는 약속이다.

  O(1) - 상수 시간 (Constant Time) : 데이터가 1개든 1억 개든 실행 시간이 똑같다.  
  O(n) - 선형 시간 (Linear Time) : 데이터 개수가 늘어나는 만큼 실행 시간도 정비례해서 늘어난다.
  

## 해시 충돌(Hash Collision)
- 서로 다른 데이터를 넣었는데 우연히 똑같은 숫자가 나올 수 있는데 이것을 해시 충돌이라고 한다  
### 파이썬의 해결책 - 오픈 어드레싱(Open Addressing) 방식을 사용한다.
- 오픈 어드레싱: "비어있는 다른 방(주소)을 찾아 떠난다"라는 뜻이다  
*특정 구역에만 몰리는 현상이 생기기 때문에 'perturb'라는 변수를 사용해 전체 테이블에 데이터를 골고루 뿌려준다.*
