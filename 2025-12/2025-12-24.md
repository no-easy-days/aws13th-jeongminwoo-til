# [2025-12-24] Git, Github


## 📝 배운 내용 (Today I Learned)

### git init
-git을 사용하기 전 저장소를 초기화한다

### git add
-커밋하고 싶은 파일을 먼저 staging area에 올린다

### git commit
-staging area의 내용을 저장소에 영구적으로 기록한다

```
# 에디터 없이 바로 메세지를 쓰려면? -m 옵션
git commit -m 'add: 로그인 기능 추가!'
```
### HEAD
-HEAD는 현재 작업 중인 위치를 가리키는 포인터이다  

  1.새 커밋을 만들면 HEAD가 가르키는 브랜치가 새 커밋으로 이동한다  
  2.HEAD 자체는 계속 그 브랜치를 가르키고 있다  
  3.HEAD는 항상 가장 최근 작업 위치를 가르키고 있다  
 

### Branch
-브랜치(Branch)는 독립적인 작업 공간을 만드는 기능이다  
-브랜치(Branch)는 포인터로 특정 커밋을 가르킨다

```
git branch
# * main        ← * 표시가 현재 브랜치
#   feature

# 새 브랜치 생성 (이동하지 않음)
git branch feature/login

# 생성과 동시에 이동
git switch -c feature/login

# 병합된 브랜치 삭제
git branch -d feature/login
```

### Merge
-병합(Merge)은 두 개의 브랜치를 하나로 합치는 것이다
```
# 1단계: 합칠 "목적지" 브랜치로 이동
git switch main

# 2단계: 가져올 브랜치를 병합
git merge feature/login
```
### Conflict(충돌)
-같은 파일의 같은 줄을 수정할 때 발생한다
## 충돌했을 때
**1단계: 원하는 내용으로 수정**

특수 기호(`<<<<`, `====`, `>>>>`)를 모두 지우고 최종적으로 원하는 내용만 남긴다

**2단계: 수정한 파일 저장 후 add**
```
git add index.html
```
**3단계: 커밋으로 병합 완료**
```
git commit -m "Merge feature/login: 환영 문구 통합"
```
### git remote 
-로컬 저장소와 원격 저장소 연결 
```
git remote add origin [https://github.com/username/my-project.git](https://github.com/username/my-project.git)
```
여기서 **origin이란** 기본 원격 저장소 이름이다

### git push 
-로컬 저장소의 커밋을 원격 저장소로 업로드를 한다
```
git pull origin main
```
### git pull 
-원격 저장소의 변경사항을 로컬로 다운하고 병합(merge)한다  
**pull = fetch(다운) + merge(병합)**
```
git pull origin main
```

### pull request
-내 브랜치의 변경사항을 main 브랜치에 합치기 전에 합칠 수 있도록 요청하는 것이다   

**<만드는 방법>**   
브랜치 -> 코드 -> 커밋 -> push -> PR생성
