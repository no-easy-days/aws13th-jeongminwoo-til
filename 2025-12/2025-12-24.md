# [2025-12-24] Git, Github


## 📝 배운 내용 (Today I Learned)

### git init
-git을 사용하기 전 저장소를 초기화한다

### git add
-커밋하고 싶은 파일을 먼저 staging area에 올린다

### git commit
-staging area의 내용을 저장소에 영구적으로 기록한다

```
# 에디터 없이 바로 메세지를 쓰려면? -m 옵션
git commit -m 'add: 로그인 기능 추가!'
```
### HEAD
-HEAD는 현재 작업 중인 위치를 가리키는 포인터이다  

  1.새 커밋을 만들면 HEAD가 가르키는 브랜치가 새 커밋으로 이동한다  
  2.HEAD 자체는 계속 그 브랜치를 가르키고 있다  
  3.HEAD는 항상 가장 최근 작업 위치를 가르키고 있다  
 

### Branch
-브랜치는 포인터로 특정 커밋을 가르킨다
