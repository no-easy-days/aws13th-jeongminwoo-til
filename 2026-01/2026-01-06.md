## HTTP Request (요청)
- 클라이언트가 서버에 보내는 메세지

< 주요 http 메서드 >
| 메서드 | 의미 | 식당 비유 | 설명 |
| ---- | ---- | ---- | ---- |
| GET | 조회 | 메뉴판 보기 | 데이터를 가져온다(읽기 전용) |
| POST | 생성 | 새 주문하기 | 새로운 데이터를 서버에 전송 |
| PUT | 전체 수정 | 주문 전체 변경 | 기존 데이터를 완전히 교체 |
| PATCH | 부분 수정 | 주문 일부 변경 | 데이터의 일부만 수정 | 
| DELETE | 삭제 | 주문 취소 | 데이터를 삭제 | 

* 여기서 GET은 데이터를 url에 담아 전송해서 주소창에 보이지만 POST는 데이터를 body에 담아 전송해서 주소창에 안보인다

### HTTP Method와 CRUD 매핑 

| CRUD | HTTP Method | 의미 | 식당 비유 |
| ---- | ---- | ---- | ---- |
| create | POST | 새로운 데이터 생성 | 새 메뉴 주문 | 
| read | GET | 데이터 조회 | 메뉴판 보기 |
| update | PUT/PATCH | 데이터 수정 | 주문 변경 | 
| delete | DELETE | 데이터 삭제 | 주문 취서 |

# REST API 
- rest api는 클라이언트(사용자)가 서버에게 " 이 데이터의 상태를 주세요"라고 요청하면 서버가 해당 데이터의 현재 상태를 표현하여 전송한다.

## URL 
- 자원(데이터,파일 등)의 위치를 알려주는 고유한 주소

< URL의 구조 >

<img width="484" height="68" alt="스크린샷 2026-01-06 오전 10 35 17" src="https://github.com/user-attachments/assets/72e38ee6-7927-47d8-b828-226fec7674dc" />

| 구성 요소 | 예시 | 설명 |
| --- | --- | --- |
| 스킴(Scheme) | https | 통신 프로토콜 |
| 호스트(Host) | api.example.com | 서버의 주소(도메인 or ip) |
| 포트(Port) | 443 | 서버의 출입문 번호 |
| 경로(Path) | /users/jmw | 자원의 위치 |
| 쿼리(Query) | ?role=admin | 추가 조건 | 
| 프래그먼트<br>(Fragment) | #profile | 페이지 내 특정 위치 | 

## Resource와 Verb 분리

<명사와 동사 분리>  
Resource : 다루고자 하는 대상(사용자, 게시글 댓글 등)  
Verb : 자원에 대해 수행할 동작(조회,생성,수정,삭제)  
ex) GET(동사) + /users(명사) = 사용자 목록 조회 

# JSON
- 데이터를 저장하고 전송하기 위한 텍스트 기반의 데이터 형식  
< 핵심 함수4가지 >

| 함수 | 방향 | 입력 | 출력 |  
| ---- | ---- | ---- | ---- |
| json.dumps() | 직렬화 | python 객체 | JSON 문자열 |
| json.dump() | 직렬화 | python 객체 | 파일에 저장 | 
| json.loads() | 역직렬화 | JSON 문자열 | python 객체 | 
| json.load() | 역직렬화 | 파일 | python 객체 |
